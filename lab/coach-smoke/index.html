Darstellung

Smoke Test / Ben global / Chloe lokal

Coach UI mit echtem KI-Durchstich

Diese Seite verbindet einen globalen Header-Coach mit einem lokalen Spot-Coach. Ben hängt oben rechts im Header, Chloe bleibt an einem konkreten Lernmoment.

Was du sofort testen kannst

  • Ben erklärt, wo du bist und was diese Seite leisten soll.
  • Chloe bleibt eng an einem lokalen Spot und gibt knappe Hilfestufen.
  • Beide Antworten laufen über denselben lokalen GitHub-Models-Proxy.

1 / Header

Ben sitzt oben rechts

Klick im Header auf den Ben-Coin. Gute Testfragen sind: Wo bin ich hier?, Was soll diese Seite tun? und Was ist mein nächster Schritt?.

npm run dev npm run coach:proxy

2 / Prompt-Schnitt

Guardrails vor Magie

Der Smoke Test prüft bewusst keine Super-KI. Wir testen erst Rollenstabilität, Kontexttreue und klare Grenzen: Ben ist global, Chloe lokal, beide bleiben knapp und erfinden nichts.

  • Ben antwortet deutsch und seitenorientiert.
  • Chloe antwortet englisch und spot-orientiert.
  • Der Proxy hält Token und Modellwahl vom Frontend fern.

3 / Local Spot

Chloe am Vergleichs-Checkpoint

Dieser Spot simuliert eine kleine Lernstelle: Vor dem nächsten Parametertest soll die lernende Person zwei Runs vergleichen und einen Trade-off benennen.

Was dieser Durchstich prüft

Wenn Ben im Header sauber erklärt, wo du gerade bist, und Chloe am Spot konkret bleibt, dann haben wir den eigentlichen Produktkern schon getroffen: globaler Kontext plus lokale Hilfe auf derselben technischen Schiene.

Danach können wir gezielt nachschärfen: feinere Sprachprofile, mehr Spots, besseres Context Packing und später echte Projekt- oder Seitenwissensquellen.
