Rich Text / Fokusmodus / Overlay DE / Gefuehrt / Coach Ben

coach_text-Panel

Ben tritt als Lese-Coach nach vorn

Das ist die leichte Leseflaeche fuer kurze Hinweise, Hilfetexte oder kontextgebundene Erklaerungen.

Ben kann als lesender Coach in den Vordergrund treten, wenn die Seite fuer einen kurzen Moment verlangsamen und genau einen sauber gerahmten Gedanken setzen soll.

Das bleibt fachlich weiter coach_text in einer staerkeren Darreichung: overlay, nicht eine neue Coach-Art.

  • Warum das wirkt: Der unscharfe Hintergrund verengt die Aufmerksamkeit und macht den Text selbst zum Ereignis.
  • Warum es persoenlicher wird: Der groessere Portrait-Coin laesst den Coach sichtbar nach vorn treten, statt in einem neutralen Dialog zu verschwinden.
  • Warum HTML wichtig ist: Rich Text gibt uns Hierarchie, Tonfall, hervorgehobene Hinweise und kompakte Listen, ohne dass der Coach wie eine Dokumentseite wirkt.

Geeignet ist das fuer kurze Rahmung, eine Schluesselunterscheidung, eine Warnung vor dem naechsten Schritt oder einen konzentrierten Reflexionsmoment.

Dadurch bleibt das Modell sauber: kind = text, presentation = overlay, optional contentStyle = rich.

Rich text / focus mode / overlay EN / Guided / Coach Chloé

coach_text panel

Chloé steps forward as a reading coach

This is the lightweight reading surface for short guidance, hints or explanation snippets.

Chloé can appear as a reading-first coach when the page should briefly slow down and guide the learner through one carefully framed idea.

This is still coach_text in a stronger presentation mode: overlay, not a new coach species.

  • Why this works: the blurred backdrop narrows attention and lets the text become the main event.
  • Why it feels personal: the larger portrait coin makes the coach visibly step forward instead of hiding inside a neutral dialog.
  • Why HTML matters: rich text gives us hierarchy, voice, highlighted notes and concise lists without turning the coach into a document page.

Good uses are short framing, one key distinction, a warning before action or a reflective pause before the next step.

That means we can keep the model clean: kind = text, presentation = overlay, optional contentStyle = rich.
